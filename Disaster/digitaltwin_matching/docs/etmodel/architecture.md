# 시스템 구성 (아키텍처)

> 한 줄 요약: ETModel은 UI, 계산 엔진, 데이터셋, 시나리오 허브가 분리된 4계층 구조로 동작합니다.

## 무엇을 할 수 있나
- ETModel·ETEngine·ETSource·MyETM 간 역할을 이해하고 배포 구조를 설계합니다.
- 데이터·계산 흐름을 파악해 로컬 개발 또는 클라우드 운영 환경을 구성합니다.

## 언제/왜 쓰나
- 신규 환경에 ETModel 스택을 설치하거나 인프라를 이중화할 때
- API 연계, 데이터 업데이트, 시나리오 저장소 커스터마이징을 계획할 때

## 사용 방법 (Step-by-step)
1. 각 컴포넌트(ETModel, ETEngine, ETSource, MyETM)의 역할과 의존성을 파악합니다.
2. API 엔드포인트와 데이터 경로를 정의해 로컬 또는 배포 환경을 구성합니다.
3. 슬라이더 입력 → API 계산 → 결과 표시 전체 흐름을 검증합니다.

## 결과 해석 (주요 지표/차트)
- 구성도, 요청-응답 시퀀스, API 호출 로그를 통해 병목 및 장애 지점을 진단합니다.

## 주의사항
- 로컬 개발 시 ETModel이 올바른 ETEngine 엔드포인트를 사용하도록 환경변수를 설정합니다.
- ETSource 버전이 ETEngine과 호환되는지, 데이터 마이그레이션이 필요한지 확인합니다.
