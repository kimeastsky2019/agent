# 페이지 로드맵

> 한 줄 요약: 전력·열·수송 등 세부 부문 페이지 작성을 위한 향후 문서화 계획을 정리합니다.

## 무엇을 할 수 있나
- 예정된 세부 페이지 목록과 우선순위를 파악해 문서 작업 일정을 수립
- 관련 팀과 협업해 필요한 데이터, 스크린샷, 검수 항목을 정의

## 언제/왜 쓰나
- ETModel 상세 문서를 단계적으로 확장할 로드맵이 필요할 때
- 담당자 배정과 산출물 검수 계획을 세울 때

## 사용 방법 (Step-by-step)
1. 우선순위에 따라 작성할 페이지를 선정하고 담당자를 지정합니다.
2. 각 페이지에 필요한 데이터·스크린샷·기능 검증 항목을 수집합니다.
3. 작성 완료 후 리뷰·번역·배포 일정을 업데이트합니다.

## 결과 해석 (주요 지표/차트)
- 진행 현황 표, 간트 차트 등을 통해 문서화 진행률과 병목을 확인합니다.

## 주의사항
- 주요 기능 업데이트가 예정되어 있다면 로드맵에 반영해 문서와 서비스 기능 간 시차를 최소화합니다.

### 작성 예정 목록
1. 전력 부문: 발전원별 입력과 제약
2. 열/건물: 히트펌프·단열 등 효율화 가정
3. 수송: 전기차·수소차 보급, 충전 인프라
4. 산업: 연료전환, 전기화, CCS 적용
5. 수소 시스템: 생산·저장·수요 연계
6. 계통/유연성: 수요반응, 저장장치, 커브 지표
7. 비용/경제성: CAPEX/OPEX/LCOE 가정
8. 데이터 내보내기/버전관리: 시나리오 diff, 재현성
